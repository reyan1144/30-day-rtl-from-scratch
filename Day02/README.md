# Day 02 – Multiplexers

## Overview
This day focuses on understanding and implementing multiplexers using different coding styles in Verilog.

---

## Topics Covered

- Concept of Multiplexers (MUX)
- Data selection using control signals
- Different RTL coding styles:
  - Dataflow (assign)
  - Behavioral (if-else)
  - Behavioral (case)

---

## Modules Implemented

### 1. 2:1 Multiplexer
- Implemented using:
  - assign statement
  - if-else block
  - case statement  

### 2. 4:1 Multiplexer
- Implemented using logic expressions  
- Verified using testbench and waveform  

---

## Key Concepts

- Multiplexer selects one input from multiple inputs  
- Select lines control data flow  
- Same logic can be implemented using different coding styles  
- Understanding coding styles is important for RTL design and interviews  

---

## What This Day Demonstrates

- Writing combinational logic in Verilog  
- Using different modeling styles  
- Reusing testbenches across implementations  
- Verifying functionality using simulation

## Key Takeaway

Multiplexers are fundamental building blocks in digital design and are widely used for data selection and routing in RTL systems.
